    Pending blocker: weekly-cycle.yml workflow runs failing¶

    Two runs attempted (25250881199, 25250948797). Both failed at:

    • Doctor ingest precheck step → exit 1
    • Generate digest step → exit 1

    Same error each time:

    TypeError: Cannot read properties of undefined (reading 'GOOGLE_SDK_NODE_LOGGING')

    Root cause: @google/genai@1.50.1 reads process.env.GOOGLE_SDK_NODE_LOGGING at module-init time. On Linux Deno (CI runners) process is not yet wired up by Deno's Node compat layer when the npm dep is being evaluated. Windows Deno is more permissive, so locally this works.

    Fixes attempted¶

    1. Commit 05f9bb6 — polyfill at top of cli/centralgauge.ts body. Did NOT work because ES module imports hoist — @google/genai was evaluated BEFORE the polyfill code ran.
    2. Commit c5458ca — split polyfill into cli/_preamble.ts (no other imports) and import "./_preamble.ts"; as the FIRST statement in centralgauge.ts. ES module hoisting puts the preamble's top-level code before @cliffy/command's import chain → before the transitive @google/genai load.
    3. Not yet verified — need to trigger the workflow on master after c5458ca and watch.
